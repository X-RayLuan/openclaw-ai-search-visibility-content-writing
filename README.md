# OpenClaw AI Search Visibility Content Writing

`ai-search-visibility-content-writing` is an OpenClaw writing skill for creating content that performs well in both classic search and AI-powered search experiences.

It is built for teams that want better **SEO and GEO content writing** without falling into low-value AI content patterns. The skill helps turn an approved content brief into an **answer-first**, **entity-clear**, **non-commodity**, and **citeable** page that is easier for both human readers and AI systems to use.

## What this skill does

This repository publishes a dedicated OpenClaw skill for **AI Search Visibility Content Writing**.

The skill is designed to help write or rewrite:

- SEO blog posts
- GEO-ready articles
- AI search visibility landing pages
- product explainers
- operational guides
- comparison pages
- FAQ-driven knowledge content

## Why it exists

Many teams treat SEO and GEO as separate systems. This skill does not.

It follows the principle that **AI search visibility is an extension of strong SEO execution**, not a separate hack layer. That means the writing system prioritizes:

- direct answers near the top of the page
- clear section hierarchy
- consistent entity naming
- structured clarity through tables, checklists, steps, and FAQ blocks when useful
- non-commodity information gain
- content that is easy to quote, extract, and cite

## Core writing model

The skill is part of the broader **OpenClaw AI SEO and GEO Workflow**, but it focuses specifically on the writing layer.

Its contract is simple:

1. Answer the core query early.
2. Build the page around real subquestions.
3. Write for information gain, not filler.
4. Make the content extractable for AI systems without turning it into format theater.
5. Keep the page genuinely useful for human readers.

## Skill scope

This skill handles the **content writing layer** only. It does not own:

- keyword discovery
- topic qualification
- technical SEO validation
- schema-only optimization work
- publishing or deployment
- live QA

It assumes an upstream brief already exists and focuses on turning that brief into strong **AI search visibility content**.
